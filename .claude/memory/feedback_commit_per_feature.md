Na elke feature direct committen en pushen naar master.

**Why:** Ron is de enige developer — PR-flow en feature branches voegen alleen overhead toe.

**How to apply:** Zodra een feature af en geverifieerd is, maak direct een commit met een duidelijke message en push naar `git push origin master`. Geen aparte branches, geen PR's.
